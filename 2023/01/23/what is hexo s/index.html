what is "hexo server" ?

发表于2023-01-23|更新于2023-01-29|教程
|阅读量:

首先介绍一下hexo博客的文件目录格式

|-- _config.yml
|-- package.json
|-- package-lock.json
|-- scaffolds
|-- source
|-- _posts
|-- public
|-- themes
|-- .gitignore
|-- node_modules
|-- .deploy_git

其中有些文件夹是后续生成的，此处暂且略过。

其次简单重温一下使用hexo框架下git的基本命令

hexo cl 清理缓存，若有public文件夹则删除之  
hexo g 根据已有的框架配置生成相应的静态文章，保存在public文件夹中
hexo d 部署静态文章，可以理解成将public中的内容上传到代码托管平台上，同时生成.deploy_git文件夹

以上内容可以理解成hexo给你提供了一个博客框架，你只需要在相应的设置文件里根据喜好进行个性化设置，然后再用hexo g命令来自动生成对应的网页代码，然后用hexo d命令将代码上传到你需要的代码托管平台上(如github、gitee等)，由此可以得出一个结论：真正用于"展现"的代码内容保存在public文件夹中(.deploy_git文件夹内容相同)。

前面铺垫了这么多，接下来引入本章的主角——hexo server

hexo server命令是用于启动本地的项目，可以理解成"本地预览"，当你进行小改动时可以快速在本地预览网页效果，那么hexo server命令是执行什么文件呢？

  1. 首先使用hexo cl命令清理本地缓存，再执行hexo server，发现网页正常加载，由此可知与public文件夹无关
  2. 不关闭本地服务端口，创建一个新的文章，刷新网页，网页成功更新此文章，同理，实时更改配置信息，网页都能实时更新
  3. 得出结论，hexo server是通过现有hexo框架实时编译生成网页的

到这好像已经结束了…

但是作为一个彻彻底底的入门者，要想做一个博客显然跟着前人做好的框架进行构建比较高效，但同时又不想拘泥于有限的自由里…当然可以通过修改public文件来高自由魔改，但将来不可避免需要使用hexo cl、hexo g、hexo d三件套来覆盖原有的public文件夹

所以要如何来自由编辑并通过hexo server快速预览呢?
